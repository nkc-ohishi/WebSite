本を買わないプログラマは成長しない。

新しいことにチャレンジしないプログラマはいらない。

ここで行うべきことは、【自分で調べて問題を解決できる能力】を養うこと。

どんなことにも興味を持ち、実践してみる。

世の中のクリエイターは皆、そうやって生きています。

コンピュータがアプリケーションを実行するまで

１．アプリはストレージに保存されている

２．アプリが起動すると、ストレージからメモリにコピーされる

３．CPUがメモリにコピーされたアプリを実行する

４．アプリが終了すればメモリにコピーされたアプリは消去される

　

用語

CPU（Cセントラル・Pプロセッシング・Uユニット。中央処理装置）

　・プログラムの命令に従って、制御、演算を行う

　

ストレージ

　・ハードディスクやSSDのこと

　・情報処理用語では、「２次記憶装置」とも呼ぶ

　・外付けのUSBメモリ等は「オフラインストレージ」とも呼ばれる

　

メモリ

　・CPUで処理するプログラムやデータを一時的に置いておく場所

　・情報処理用語では、「主記憶装置・１次記憶装置」とも呼ぶ

アプリケーション（実行ファイル）ができるまで

・人間がプログラム言語を利用して命令書を作る

・人間が作った命令書をコンピュータが理解できる命令書に変換する（コンパイル）

・変換された命令書のことをアプリケーション（実行ファイル）と呼ぶ

　

実行ファイル

Windows　→　.exe

Mac　→　.app

Android　→　.apk

iOS　→　.ipa

　

※ コンピュータが理解できる命令書を作成するアプリケーションがVisualStudio

C#を勉強する上で最低限、覚えなければならない用語

クラス

　・スクリプトの構成単位。Unityでは基本1スクリプト1ファイル。

　・ファイル名とクラス名は同じでなければならない。

　

変数

　・プログラム上で、何かの値を名前を付けて保存するすることができる入れ物のようなもの。

　

メソッド

　・プログラマが命令した通りに、何かしらの仕事をしてくれる頼もしい奴ら。

　

引数

　・プログラマが、メソッドに仕事を依頼するときに渡す値（情報）。ない場合もある。

　

戻り値

　・プログラマが、メソッドに仕事を依頼し、仕事をした結果の報告値。ない場合もある。

　

実行ファイル

Windows　→　.exe

Mac　→　.app

Android　→　.apk

iOS　→　.ipa

　

※　この授業ではざっくり説明するので、詳しくはプログラミング技法の先生に聞いてください。

いちからコードを書いてみる

大前提

　・C#プログラムは、メソッドの組み合わせでできている

　

メソッドのテンプレート

　戻り値の型　メソッド名（引数）

　{

　　仕事の内容

　}

　

・既存のメソッドを使ったり、新しいメソッドを作ったりしてプログラムを構築する

　

・C#プログラムはMainメソッドから開始される

　　(Unityはいろいろな処理が秘密裏に行われており、Mainメソッドは隠れている)

　

手順１．Mainメソッドを用意する

　

　　仕様

　　メソッド名　Main（決まっている）

　　引　数　　　なし

　　戻り値　　　なし

　

　　void Main()

　　{

　

　　}

※ C#には様々なルールがあり、これだけではエラーになる。

　

手順２．ルールその１．「メソッドはクラス内に書かなければいけない」

　　　　↓

　　　　ファイル名と同じ名前のクラスを作り、クラスのブロック内にMainメソッドを記述する。

　　　　例）

　　　　class Program

　　　　{

　　　　　　void Main()

　　　　　　{

　

　　　　　　}

　　　　}

　

手順３．ルールその２．「Mainメソッドは静的メソッドでなければならない」

　　　　↓

　　　　戻り値の型の前に「static」をつける

　　　　例）

　　　　class Program

　　　　{

　　　　　　static void Main()

　　　　　　{

　

　　　　　　}

　　　　}

　　　　※ これでエラーが無くなるはず。（最小のC#プログラム）
